# 如何通过 GitHub 项目降低 AI 生成文本的 AI 率

本次改写先固定材料边界：我只使用`examples/source_brief.md`、`examples/original.md`、`examples/revised.md`、`adversarial_loop.py`、`auto-de-ai-writing-skill`和来源说明中的项目需求。没有依据的经历、引用和第三方检测结果不写入正文。

AI 生成文本正在改变用户的写作方式和内容生产方式。在本次材料中，本次要验证的是如何降低 AI 生成文字的 AI 率。通过 GitHub 项目、skills 以及相关自动化方法，可以降低本地 AI-like Rate，并在报告中展示指标变化。

可以利用 AI 工具对生成文本进行按模板表达、空泛词、连接词和作者痕迹等指标分析，从而发现文本中存在的问题。可以通过自动化程序对文本按风险句子修改，使语言表达更加自然，段落围绕来源、实现和限制展开。还可以结合不同的检测方式，对修改前后的结果进行对比，从而复查修改前后的指标变化。

在具体实现过程中，GitHub 项目可以承担版本记录和流程复现。它能够保存完整的处理流程；它也能够展示项目的技术特点和可复现结果。通过构建一个 skill，可以实现从文本检测、内容分析到自动改写的完整流程。这种方式会保留可复现的输入、输出和限制说明。

AI 生成文本的降重需要同时检查表达方式、逻辑结构和内容质量。只有在合理使用 AI 工具的基础上，才能更好地降低 AI 率，并避免文本过于机械化和模板化的问题。自动化降 AI 率方法具有一定的实验指标意义。

通过构建 GitHub 项目和 skill，可以形成从检测、分析、改写到报告生成的完整闭环。该方法能够降低本地 AI-like Rate，并充分体现文本具体化效果，在中文写作辅助项目场景下可以作为可检查的展示案例。

检测分数只作为项目内的实验指标，不能证明文本一定由 AI 或人类撰写。
